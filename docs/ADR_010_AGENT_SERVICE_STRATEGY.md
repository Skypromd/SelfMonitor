# ADR 010: канонический AI-ассистент (GraphQL / сервис)

## Статус

Принято (2026-04-10).

## Контекст

В репозитории встречаются **`agent-service`** и **`ai-agent-service`**. GraphQL gateway в optional profile не должен ссылаться на «пустые» имена субграфов.

## Решение

- **Канонический продуктовый сервис ассистента:** **`agent-service`** (основной REST/интеграционный контур в compose).
- **`ai-agent-service`** трактуется как экспериментальный / legacy alias: новые маршруты и документация добавляются к **`agent-service`**, пока не будет явной миграции.
- Federation: при включении субграфов env URL должны указывать на **один** выбранный сервис; дублирование одной и той же схемы под двумя именами запрещено.

## Последствия

- CI/compose/доки выравниваются постепенно под `agent-service`.
- Маркетинг: не обещать отдельный «AI Agent Service» как второй продукт без изменения ADR.
