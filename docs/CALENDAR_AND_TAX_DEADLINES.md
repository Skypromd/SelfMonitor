# Календарь и налоговые сроки

## Текущее состояние

- **`calendar-service`**: локальное хранилище событий пользователя (`calendar_events`), JWT-аутентификация.
- **`tax-engine`**: расчёты и интеграция с календарём через `CALENDAR_SERVICE_URL` (создание событий при необходимости).

## Целевой контур

- Единый источник **обязательств HMRC** (кварталы, финальная декларация) — `tax-engine` + конфиг/данные периодов.
- **calendar-service** получает дедлайны (push или pull) и отдаёт напоминания в UI; **без** автоматической отправки в HMRC на низких тарифах.

## Тесты

- Контракт: ответ tax-engine с датами периода согласован с MTD-календарём; тесты в `tax-engine` / интеграционные — по мере расширения.
