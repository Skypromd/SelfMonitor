# MyNetTax — Brand Book (v1)

Живой документ: обновляйте версию и дату при существенных изменениях. Дизайн в продукте опирается на `apps/web-portal/styles/tokens.css` и **Inter**.

---

## 1. Суть бренда

### 1.1 Обещание (brand promise)

**«Чёткая финансовая и налоговая картина для самозанятых в UK — в одном месте, без хаоса таблиц и дедлайнов.»**

Не «мы заменяем бухгалтера», а «мы даём порядок, прозрачность и уверенность перед HMRC и банком».

### 1.2 Опоры (pillars)

| Столп | Что это значит для копирайта и UI |
|--------|-----------------------------------|
| **Ясность** | Меньше жаргона, больше «что это значит для вас» и сумм в фунтах. |
| **Контроль** | Пользователь явно подтверждает действия (синк банка, отправка в HMRC). |
| **Справедливость** | Честные тарифы, понятные лимиты, без «тёмных» списаний. |
| **Уважение к UK-контексту** | MTD, CIS, self-employed — нативно, без «перевода с американского». |

### 1.3 Аудитории (приоритеты)

1. Самозанятые и малый бизнес (в т.ч. **мигранты** с несколькими банками — сильная сторона Salt Edge в продукте).
2. CIS-подрядчики, которым важны **доказательства** и возвраты.
3. Те, кто готовится к **ипотеке** (готовность к документам и стабильному доходу).

## 2. Имя, аббревиатура, домены

| Элемент | Правило |
|---------|---------|
| **Полное имя** | **MyNetTax** — так в договорах, футере, письмах, HMRC product strings, App Store listing. |
| **Аббревиатура** | **MT** — на марке и в компактном UI (иконка, топ-бар, splash); полное имя **MyNetTax** золотом под строкой MT в логотипе. |
| **Чтение** | Полное: «Май Нет Такс»; на марке сверху только **MT** (My + Tax), без точек. |
| **Домены** | Канонический публичный сайт выберите один (`mynettax.com` или `mynettax.co.uk`) и закрепите в конфиге; остальные — 301 на канон. `mynettax.app` — удачно для приложения и коротких ссылок. |

**Нарратив имени:** «Net» = связь счетов, потоков, данных в одну сеть; «Tax» = налоговый фокус без размывания в «ещё одном neobank».

## 3. Логотип

### 3.1 Состав системы

Рекомендуется иметь **три уровня** (дизайнер готовит в Figma / Illustrator):

1. **Символ** — квадратный марк для favicon, аватаров, splash (`public/branding/mynettax-icon.svg`): сверху **MT** (бирюза), снизу мелко **MyNetTax** (золото).
2. **Горизонтальный логотип** — символ + слово **MyNetTax** (для сайта, презентаций, email header).
3. **Вордмарк** — только **MyNetTax** (где символ уже есть рядом или мешает).

### 3.2 Направления для улучшения марка (идеи, не финал)

- **Сеть + налог:** абстрактный узел из трёх точек (M-N-T или три «счёта») тонкой линией; избегать «щита HMRC» и гербов.
- **Точность:** лёгкий «tick» или выравнивающая линия внутри буквы **T** — ассоциация с проверкой, не с галочкой «скидка».
- **Монограмма MT:** две буквы из **одной толщины штриха** (как SF Pro / Inter Black), строка MyNetTax подписью; в финале — **outline** в SVG, без зависимости от шрифта в браузере и на печати.
- **Светлая тема:** инверсия — тёмный знак на белом для PDF счетов и печати.

### 3.3 Правила использования

| Делать | Не делать |
|--------|-----------|
| Сохранять пропорции и отступы (clear space ≥ высоты буквы **N** вокруг символа). | Растягивать, поворачивать, добавлять тени и свечения «ради красоты». |
| На фото — монохром или белая версия на тёмном фоне. | Класть логотип на «шумный» фон без подложки. |
| Для очень малых размеров (<18px) — упрощённый символ без мелких деталей. | Подменять шрифт вордмарка на декоративный «финтех-пузырь». |

### 3.4 Минимальные размеры (ориентир)

- **Favicon / app icon:** не мельче **16×16** логически; для legibility символа лучше **24px+**.
- **Печать визитки:** вордмарк не уже **22mm** по ширине.

## 6. Тон коммуникации (tone of voice)

- **Спокойный эксперт**, не паникёр: «Дедлайн через 14 дней» вместо «СРОЧНО!!!».
- **Конкретика:** даты, суммы, шаги 1–2–3.
- **Уважение к риску:** формулировки в духе «HMRC может запросить…», без запугивания.
- **Инклюзивность:** UK English по умолчанию; мультиязычность — как сильная сторона, не «дополнение».

**Слова-якоря:** clarity, control, compliant (в смысле процесса), one place.

**Избегать:** «полная автоматизация налога без вашего участия», обещания «одобрения ипотеки», визуального смешения с HMRC.

---

## 7. Иконки, иллюстрации, фото

- Иконки: **Lucide**-совместимый стиль — тонкий штрих, скругления как в UI.
- Иллюстрации: плоские или мягкий 3D, **без стоковых «рукопожатий в офисе»**; лучше абстрактные схемы потока денег.
- Фото людей: реалистичные ситуации (ноутбук, кофе, стройка для CIS), **согласие на использование**.

## 8. Бизнес-задумки и активации бренда

Идеи, которые **усиливают** MyNetTax, а не размывают продукт:

1. **«Налоговая гигиена» как рубрика** — короткие чек-листы перед концом налогового года; бренд = порядок, не страх.
2. **Партнёрство с бухгалтерами** — white-label отчёт «MyNetTax export for accountant»; брендированный PDF пакет.
3. **CIS как герой-история** — кейсы «возврат / доказательства»; отдельный визуальный подпакет (цветовой акцент можно чуть смещать, но в рамках палитры).
4. **Календарь дедлайнов** — публичная страница + подписка на email от **MyNetTax**; виральность без спама.
5. **Реферальная механика** — «пригласи коллегу по CIS»; сообщение уже с доменом `mynettax.app` — держать один CTA.
6. **Сертификации и доверие** — в маркетинге только **проверяемые** формулировки (FCA disclaimer у вас уже в духе «не совет» — не ослаблять).
7. **Merch / конференции** (на будущее) — марка **MT** + полное имя на обороте визитки; не смешивать с логотипами банков на одном носителе без разрешения.

## 9. Юридическое и комплаенс

- Не использовать цвета/форму, создающие **путаницу с HMRC** или «государственное одобрение».
- В письмах и футере: **юридическое имя компании** и регистрация (как в Terms).
- Product name в API HMRC — согласовать с `HMRC_VENDOR_PRODUCT_NAME` и не менять без регистрации нового продукта.

### 9.1 Инфраструктура после ребренда

- Во всех сервисах одинаково задайте **`INTERNAL_JWT_AUDIENCE=mynettax-internal`** (и `libs/shared_auth` по умолчанию совпадает). Старый `selfmonitor-internal` больше не используется в шаблонах compose / `.env.example`.
- **Salt Edge:** префикс клиента по умолчанию `mynettax`; схема deep link в мобильном приложении пока **`selfmonitor://`** — меняйте только вместе с `app.json` и регистрацией redirect в Salt Edge.
- **Android package** в ссылках на Play Market может оставаться `com.selfmonitor.app`, пока не выпущено новое приложение.
