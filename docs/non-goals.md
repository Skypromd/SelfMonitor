# Non-goals для production v1

Список того, что **намеренно не входит** в первый продакшен‑релиз SelfMonitor (чтобы не раздувать scope и сроки).

## Продукт и фичи

- **Голосовой ассистент end-to-end в prod** (`voice-gateway` как обязательная часть продукта) — позже, отдельный релиз и ключи.
- **Полноценный BI / отдельный business-intelligence** как обещание v1 — отчёты ипотеки/аналитика остаются в **analytics-service** в рамках плана.
- **Marketplace партнёров / white-label** как обязательный контур v1.
- **Мульти-тенант / team seats** на одну подписку — политика: **один пользователь на подписку** (`AGENTS.md`, `PLAN_FEATURES`).
- **Авто-submit в HMRC** без явного подтверждения пользователя — запрещено политикой (`docs/POLICY_SPEC.md`).

## Технология и инфраструктура

- **GraphQL Federation** для всех клиентов — REST через gateway достаточен для v1; GraphQL — профиль `graphql`.
- **Полный service mesh (mTLS везде)** — roadmap; v1: секреты, WAF‑паттерны nginx, rate limits на auth/billing/upload/api-key.
- **Идеальный OpenTelemetry на всех сервисах** — внедрять поэтапно после v1 ядра.
- **Антивирус на каждый upload в v1** — желательно до масштаба; пока MIME/размер/частные бакеты (см. runbooks).

## Комплаенс и процессы

- **Полный pen-test** до первого soft launch — не блокер для закрытого пилота; до публичного маркетинга — минимум OWASP ZAP / внешний скан.
- **Автоматический GDPR data portability** как полный self-service — может быть запрос в support в v1.

## Документ как контракт

Изменения **non-goals** только через явное решение продукта/безопасности; обновляйте дату внизу.
