# Threat model (кратко)

SelfMonitor — B2C UK fintech: банковские данные, документы, налоговые сведения, JWT-доступ к API.

## Основные активы

- Учётные данные и сессии пользователей (`auth-service`).
- Транзакции и PII (`transactions-service`, `banking-connector`).
- Документы и OCR (`documents-service`, S3-совместимое хранилище).
- Налоговые сценарии и отправки HMRC (`tax-engine`, `integrations-service`).

## Доверенные границы

- Клиенты (web, mobile) → **nginx** `:8000` → внутренние сервисы.
- Секреты только из env / secret manager в проде; не в git и не в образах как константы.

## Угрозы и смягчение

| Угроза | Риск | Меры |
|--------|------|------|
| Утечка JWT / кража сессии | Высокий | HTTPS, короткий TTL refresh, хранение секрета, ограничение CORS в проде |
| Обход тарифных лимитов | Средний | Проверки лимитов в API (`plan_limits`, счётчики) |
| Подделка HMRC submit | Высокий | Двухфазное подтверждение (`HMRC_REQUIRE_EXPLICIT_CONFIRM`), аудит токенов |
| OCR / документы: malware upload | Средний | Ограничение типов/размера, изолированная обработка |
| SSRF из сервисов к внутренним URL | Средний | Белые списки URL, не доверять пользовательским хостам |
| Недоступность зависимостей | Операционный | Health + алерты, см. `docs/SECURITY_CHECKLIST.md` |

Документ обновлять при смене архитектуры или новых интеграциях.
