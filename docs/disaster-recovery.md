# Disaster recovery (lite)

Цели **v1** зафиксированы в **`docs/production-scope.md`** (черновик RPO/RTO); ниже — общий процесс.

## Что бэкапим

- **PostgreSQL** (все БД сервисов на Postgres): ежедневный логический дамп или снапшот провайдера.
- **Объектное хранилище** (документы): версионирование / cross-region по политике продукта.
- **SQLite** (если остаётся в каких-то сервисах dev/demo): не опора для prod; мигрировать на Postgres для prod.

## Восстановление

Пошагово: **`docs/runbooks/restore-db.md`**.

## Проверки

- Раз в квартал (минимум): **restore drill** на изолированный кластер или staging.
- Фиксировать дату и результат в тикете/доке.

## Связь с алертами

- При потере бэкапов или сбое репликации — P1 по внутреннему регламенту.
