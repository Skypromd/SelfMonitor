# Bank Sync Economics — Оптимизация частоты синхронизации по тарифам

## Вводные

- Open Banking провайдер: SaltEdge (usage-based)
- Каждый **refresh** (обновление данных с банка) = 1 API call = деньги
- **Модель: sync ТОЛЬКО по активации** — нет фоновой автосинхронизации
- Юзер сам нажимает кнопку "Sync" или получает push-оповещение и заходит
- Лимит sync/день зависит от тарифа: 0 / 1 / 2 / 3
- Не зашёл в приложение = 0 расход

## Финальная модель: Sync по активации ✅

**Принцип**: банк обновляется ТОЛЬКО когда юзер сам нажал "Sync" или зашёл по push-оповещению. Нет фоновой синхронизации. Не зашёл = £0 расход.

### Тарифы и лимиты sync

| Тариф | Sync лимит/день | Bank sync | Оповещения |
|---|---|---|---|
| **Free £0** | **0** (нет sync) | ❌ Ввод транзакций вручную | "Connect your bank — upgrade to Starter" |
| **Starter £9** | **1×/день** | ✅ Кнопка Sync, 1 банк | Push утром: "☀️ Tap to sync your bank" |
| **Growth £12** | **2×/день** | ✅ Все банки | Push утро + вечер: "🌙 Evening sync ready" |
| **Pro £15** | **3×/день** | ✅ Все банки | Push утро + обед + вечер |
| **Business £25** | **3×/день** + instant push | ✅ Все банки | Push при каждом входе: "Tap to get latest" |

### Реальная активность юзеров (DAU)

Не все 100K юзеров заходят каждый день:

| Метрика | Значение |
|---|---|
| DAU/MAU (daily active / monthly active) | 25-35% для финтеха |
| Среднее посещений у активного юзера | 1.3-1.8 раз/день |
| Юзеры не заходящие в течение месяца | 15-20% |

**В любой день заходят ~30% юзеров, не 100%.**

### Сравнение: авто-sync vs по активации

| | Авто-sync (фоновый) | **По активации** |
|---|---|---|
| Refreshes/мес | 24,050,000 | **1,102,000** |
| Разница | | **в 22 раза меньше** |
| Стоимость (£0.005/refresh) | £120,000/мес | **£5,510/мес** |
| % от revenue (£725K) | 16.6% | **0.76%** |
| Экономия | — | **£114,490/мес** |

### Почему модель "по активации" лучше

1. **£5.5K вместо £120K** — экономия £1.37M/год
2. **Free юзеры стоят £0** — 40K юзеров, zero cost, максимальный upgrade pressure
3. **Неактивные юзеры не тратят** — нет логина = нет refresh = нет расхода
4. **Push-оповещения = retention** — юзер получает push "Sync available" → заходит → использует продукт
5. **Простой upgrade path** — "Want 2 syncs/day? Upgrade to Growth"
