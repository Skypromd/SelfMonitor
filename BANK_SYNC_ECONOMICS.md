# Bank Sync Economics — Оптимизация частоты синхронизации по тарифам

## Вводные

- Open Banking провайдер: SaltEdge (usage-based)
- Каждый **refresh** (обновление данных с банка) = 1 API call = деньги
- Цель: максимальный perceived value для юзера при минимальных расходах

## Почему Вариант C оптимален

### Для Free юзера: ручной sync (5×/мес)
- **Что видит**: кнопка "🔄 Sync bank" на дашборде, счётчик "4 syncs remaining"
- **Почему работает**: юзер получает ценность (видит транзакции), но чувствует ограничение
- **Upgrade trigger**: "Your bank data is 3 days old. Upgrade to Starter for daily auto-sync"
- **Стоимость**: £1K/мес на 40K юзеров = £0.025/юзер/мес → практически бесплатно

### Для Starter (£9): 1×/день
- **Что видит**: транзакции обновляются каждое утро в 6:00
- **Почему работает**: 90% самозанятых проверяют финансы 1 раз в день — этого достаточно
- **Воспринимаемая ценность**: "Каждое утро открываю — всё актуально"
- **Стоимость**: £6.8K/мес на 30K юзеров = £0.23/юзер/мес → при £9 ARPU = 2.5% расход

### Для Growth (£12): 3×/день
- **Что видит**: обновление утром (7:00), днём (13:00), вечером (19:00)
- **Почему работает**: ловит утренние покупки к обеду, обеденные — к вечеру
- **Upgrade trigger**: "See transactions within 2 hours — upgrade to Pro"
- **Стоимость**: £13.5K/мес на 15K юзеров = £0.90/юзер/мес → при £12 ARPU = 7.5% расход

### Для Pro (£15): каждые 2 часа
- **Что видит**: транзакции появляются с задержкой максимум 2 часа
- **Почему работает**: для активных фрилансеров с 5+ транзакций/день
- **Killer feature**: push-уведомление "New transaction: -£50.00 at Shell"
- **Стоимость**: £45K/мес на 10K юзеров = £4.50/юзер/мес → при £15 ARPU = 30% расход

### Для Business (£25): каждый час + on-demand
- **Что видит**: практически real-time + кнопка "Sync now" без лимита
- **Почему работает**: business юзеры управляют cash flow в реальном времени
- **Premium feel**: "I always know exactly how much I have"
- **Стоимость**: £54K/мес на 5K юзеров = £10.80/юзер/мес → при £25 ARPU = 43% расход

## Маржинальность по тарифу

| Тариф | ARPU | Sync cost/юзер | Payment cost (GC) | Infra/юзер | Margin/юзер | % margin |
|---|---|---|---|---|---|---|
| **Free** | £0 | £0.025 | £0 | £0.05 | **-£0.075** | — (acquisition cost) |
| **Starter** | £9 | £0.23 | £0.09 | £0.15 | **£8.53** | **94.8%** |
| **Growth** | £12 | £0.90 | £0.12 | £0.20 | **£10.78** | **89.8%** |
| **Pro** | £15 | £4.50 | £0.15 | £0.25 | **£10.10** | **67.3%** |
| **Business** | £25 | £10.80 | £0.25 | £0.40 | **£13.55** | **54.2%** |

### Выводы

1. **Starter — самый маржинальный** (94.8%). Именно этот тариф надо продвигать
2. **Free стоит почти ничего** — £0.075/юзер/мес. 40K free юзеров = £3K/мес → отличная цена за потенциальную конверсию
3. **Pro/Business дорогие по sync** — но эти юзеры платят больше и имеют выше LTV
4. **Blended margin по sync**: £120K cost / £725K revenue = **16.6%** — здоровый показатель

## Оптимизации для снижения costs

| Оптимизация | Экономия | Сложность |
|---|---|---|
| **Smart scheduling**: sync только в рабочие часы (8-22), не ночью | -30% refreshes | Низкая |
| **Delta sync**: если банк не менялся — skip refresh | -20% refreshes | Средняя |
| **Weekend reduction**: 1×/день в выходные для всех кроме Business | -15% refreshes | Низкая |
| **Inactive user pause**: нет логина 7 дней → pause sync | -10-15% refreshes | Низкая |
| **Batch refresh**: sync все банки юзера одним вызовом | Зависит от провайдера | Средняя |

С оптимизациями: £120K → **~£70-80K/мес** (10-11% от revenue)
